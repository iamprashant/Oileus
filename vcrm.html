VCRM - View Controller Remote Model is a design pattern, which is followed by all Oileus applications. It has minor difference from MVC. Here the model resides at remote location, controller and view reside at local. The model can be written in any language, but for communication it understands JSON only.

Controller

In Oileus controller is the devision of code, which focus on specific module or sub module. For defining new controller in Oileus, this syntax can be followed-

oileus.controller("MyController", function() { //constructor code });

Action

In oileus all the member of controllers are called Action, which act for some task. If a controller is already defined the actions can be added in following manner-

oileus.actFor("MyController", "myAction", function(arg1, arg2) { //code });

Note - If the controller is not defined it adds the controller automatically.

Process

Processes are special type of function, which become active when they recieve model of given type. For example if you have defined process listener for Box, then if the data recieved from model contains the Box type, it automatically become active and performs it's default behaviour. For defining process we can follow this syntax-

/** * Example info structure is: * {status, message} */ oileus.process("myDynamicBox", function(info) { if(info.status === "success") { //code to display the message alert(info.message); } else { alert("Something went wrong!"); } });

Note - There are two default process handlers are there - link and script

Key points

  • Process listeners should return boolean. If you want to skip further processing return false else true.
  • If you want to access response, add one argument to method
  • The name given to process should be unique
  • If you want to override the process, you can add new process with the same name
